## 발표자료 생성 요청 문장

업로드 후 NotebookLM에 다음처럼 요청한다.

```text
업로드한 자료를 기반으로 10분 발표자료 초안을 만들어줘.
슬라이드는 9장 내외로 구성하고, 각 슬라이드마다 제목, 핵심 bullet 3개 이내, 발표자 노트 3~5문장, 넣으면 좋은 표/코드/결과를 제안해줘.
핵심 메시지는 L2 -> L3/L4 -> SA로 갈수록 AI 생성 코드가 유스케이스 흐름 중심에서 도메인 객체, 책임 경계, 성능 인덱스, 동시성 임계 영역을 포함하는 구조로 변화했다는 점이야.
특히 동시 입찰/동시 낙찰 경로는 auctionId 단위 임계 영역으로 보호되며, 단일 JVM 멀티스레드 테스트에서 thread-safe하게 동작함을 확인했다는 점을 강조해줘.
톤은 과장된 마케팅이 아니라 기술 발표처럼 명확하고 방어 가능하게 작성해줘.
```

## 발표 핵심 문장

```text
L2는 유스케이스 흐름을 코드화했지만 동시 요청 상황에서는 check-then-save 경쟁 조건이 남았다.
L3/L4는 책임 경계를 SubmitBidManager, SelectWinnerManager로 구조화했다.
SA는 성능과 동시성 품질 기준을 명시하여, 단일 JVM 멀티스레드 테스트에서 핵심 입찰/낙찰 경로가 thread-safe하게 동작함을 확인했다.
```
