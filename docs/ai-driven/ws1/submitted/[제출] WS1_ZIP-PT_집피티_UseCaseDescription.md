## 2-1. Use Case Description — AI 생성본

**[작성 규칙]**
- 필드 순서: Use Case Name → Summary → Actor → Dependency → Precondition → Description → Alternatives → Postcondition
- Use Case Name: 동사+명사 형태
- Summary: 주어+동사+목적어 단일 문장
- Actor: 직접 시작하는 외부 행위자만 명시
- Dependency: «include» 관계 시에만 기재
- Precondition: 구체적 시스템/데이터 상태로 서술
- Description: 정상 흐름 각 단계를 번호 + 단일 행동 단위로 기술
- Alternatives: [단계번호] 표기와 함께 기술
- Postcondition: 정상/비정상 종료 구분하여 상태로 서술

### UC-01. Register Auction (매물 경매 등록) — AI 생성본

| 필드 | 내용 |
|---|---|
| **Use Case Name** | Register Auction (매물 경매 등록) |
| **Summary** | 매도자가 보유 매물에 대한 역경매 공고를 시스템에 등록한다. |
| **Actor** | 매도자 (Seller) |
| **Dependency** | «include» Validate User |

**Precondition**

- P1. 매도자는 시스템에 로그인되어 인증 세션이 유효한 상태이다.
- P2. 등록할 매물 정보(주소, 면적, 희망 매도가)가 시스템에 사전 입력된 상태이다.
- P3. 해당 매물에 대해 진행 중인 경매 공고가 없는 상태이다.

**Description (정상 흐름)**

1. 시스템은 Validate User를 수행하여 매도자의 인증 상태를 확인한다.
2. 매도자는 경매 공고 등록을 요청한다.
3. 시스템은 매도자 소유 매물 목록을 반환한다.
4. 매도자는 공고 대상 매물을 선택한다.
5. 시스템은 선택된 매물의 기본 정보를 공고 등록 양식에 자동으로 채운다.
6. 매도자는 공고 조건(입찰 마감 기한, 요구 서비스 조건, 최소 자격 요건)을 입력한다.
7. 시스템은 입력된 공고 조건의 유효성(마감 기한 범위, 필수 항목 충족)을 검증한다.
8. 매도자는 공고 최종 등록을 확정한다.
9. 시스템은 공고를 경매 저장소에 저장하고 공고 ID를 발급한다.
10. 시스템은 공고 상태를 '입찰 모집중'으로 설정한다.
11. 시스템은 매도자에게 공고 등록 완료 메시지와 공고 ID를 반환한다.
12. 시스템은 공고 목록 화면으로 복귀하여 정상 상태로 종료한다.

**Alternatives**

**A1. [4단계] 선택한 매물에 이미 진행 중인 공고가 있는 경우**
- A1-1. 시스템은 '해당 매물에 진행 중인 공고가 존재합니다' 메시지를 표시한다.
- A1-2. 시스템은 매물 선택 화면(3단계)으로 복귀한다.

**A2. [7단계] 공고 조건 유효성 검증 실패 시**
- A2-1. 시스템은 오류 필드를 강조 표시하고 수정 안내 메시지를 출력한다.
- A2-2. 매도자는 6단계로 돌아가 공고 조건을 재입력한다.

**A3. [Cancel — 6~8단계] 매도자가 공고 등록을 취소한 경우**
- A3-1. 시스템은 임시 입력 데이터를 파기한다.
- A3-2. 시스템은 공고 목록 화면으로 복귀하며, 경매 저장소에는 어떠한 변경도 가하지 않는다.

**Postcondition**

- Q1. (정상 종료) 신규 경매 공고가 경매 저장소에 저장되며, 공고 상태는 '입찰 모집중'이다.
- Q2. (비정상 종료 — A1/A2/A3) 경매 저장소 상태는 Precondition 시점과 동일하게 유지된다.

### UC-02. Submit Bid (입찰 참여) — AI 생성본

| 필드 | 내용 |
|---|---|
| **Use Case Name** | Submit Bid (입찰 참여) |
| **Summary** | 공인중개사가 매도자의 역경매 공고에 대해 수수료·서비스 조건을 포함한 입찰서를 제출한다. |
| **Actor** | 공인중개사 (Agent) |
| **Dependency** | «include» Validate User |

**Precondition**

- P1. 공인중개사는 시스템에 로그인되어 인증 세션이 유효한 상태이다.
- P2. 대상 경매 공고가 '입찰 모집중' 상태이며 입찰 마감 시각 이전이다.
- P3. 공인중개사의 자격 증명(자격증 번호, 사무소 등록증)이 검증 완료된 상태이다.
- P4. 공인중개사는 해당 공고에 대해 아직 입찰서를 제출하지 않았거나 재제출이 허용된 상태이다.

**Description (정상 흐름)**

1. 시스템은 Validate User를 수행하여 공인중개사의 인증 상태를 확인한다.
2. 공인중개사는 참여 가능한 경매 공고 목록 조회를 요청한다.
3. 시스템은 마감 시각이 경과하지 않은 활성 공고 목록을 반환한다.
4. 공인중개사는 입찰할 공고를 선택한다.
5. 시스템은 선택된 공고의 상세 정보(매물 정보, 매도자 요구 조건, 현재 입찰 건수)를 표시한다.
6. 공인중개사는 입찰서(제안 수수료율, 마케팅 전략, 예상 매각 기간, 서비스 조건)를 작성한다.
7. 시스템은 입력된 입찰 정보의 유효성(필수 항목 충족, 수수료율 범위, 글자 수 제한)을 검증한다.
8. 공인중개사는 최종 입찰서 제출을 확정한다.
9. 시스템은 입찰서를 경매 저장소에 저장하고 접수 시각을 기록한다.
10. 시스템은 공인중개사에게 입찰 접수 완료 메시지와 접수번호를 반환한다.
11. 시스템은 매도자의 알림 큐에 신규 입찰 알림을 등재한다.
12. 시스템은 입찰 목록 화면으로 복귀하여 정상 상태로 종료한다.

**Alternatives**

**A1. [4단계] 선택한 공고가 이미 마감된 경우**
- A1-1. 시스템은 '공고가 마감되었습니다' 메시지를 표시한다.
- A1-2. 시스템은 공고 목록(3단계)으로 복귀한다.

**A2. [7단계] 입력 유효성 검증 실패 시**
- A2-1. 시스템은 오류 필드를 강조 표시하고 수정 메시지를 출력한다.
- A2-2. 공인중개사는 6단계로 돌아가 입찰서를 재작성한다.

**A3. [Cancel — 6~8단계] 공인중개사가 입찰 작성을 취소한 경우**
- A3-1. 시스템은 임시 입력 데이터를 파기한다.
- A3-2. 시스템은 공고 목록(3단계)으로 복귀하며, 경매 저장소에는 어떠한 변경도 가하지 않는다.

**A4. [9단계] 저장 실패 시**
- A4-1. 시스템은 처리를 취소하고 이전 상태로 복원한다.
- A4-2. 시스템은 '일시적 오류로 저장되지 않았습니다' 메시지를 반환하고 6단계로 복귀한다.
- A4-3. 시스템은 장애 로그에 실패 이벤트를 기록한다.

**Postcondition**

- Q1. (정상 종료) 입찰서가 경매 저장소에 저장되며, 해당 공고의 입찰 건수가 1 증가한 상태이다.
- Q2. (정상 종료) 매도자의 알림 큐에 입찰 알림 메시지가 등재된 상태이다.
- Q3. (비정상 종료 — A1/A2/A3) 경매 저장소 및 알림 큐 상태는 Precondition 시점과 동일하게 유지된다.
- Q4. (비정상 종료 — A4) 경매 저장소는 Precondition 상태로 복원되고, 장애 로그에 실패 이벤트가 1건 추가된 상태이다.

### UC-03. Select Winner (낙찰자 선정) — AI 생성본

| 필드 | 내용 |
|---|---|
| **Use Case Name** | Select Winner (낙찰자 선정) |
| **Summary** | 매도자가 입찰 마감된 경매 공고에서 낙찰 공인중개사를 선정한다. |
| **Actor** | 매도자 (Seller) |
| **Dependency** | «include» Validate User / «extend» Submit Bid |

**Precondition**

- P1. 매도자는 시스템에 로그인되어 인증 세션이 유효한 상태이다.
- P2. 대상 경매 공고의 입찰 마감 시각이 경과한 상태이다.
- P3. 해당 공고에 1건 이상의 입찰서가 접수된 상태이다.
- P4. 해당 공고에 대해 아직 낙찰자가 선정되지 않은 상태이다.

**Description (정상 흐름)**

1. 시스템은 Validate User를 수행하여 매도자의 인증 상태를 확인한다.
2. 매도자는 마감된 경매 공고 목록 조회를 요청한다.
3. 시스템은 입찰 마감되고 낙찰자 미선정 상태인 공고 목록을 반환한다.
4. 매도자는 낙찰자를 선정할 공고를 선택한다.
5. 시스템은 해당 공고에 접수된 입찰서 목록(수수료율, 마케팅 전략, 서비스 조건, 평판 점수)을 표시한다.
6. 매도자는 입찰서를 비교 검토한다.
7. 매도자는 최종 낙찰할 공인중개사를 선택하고 낙찰 확정을 요청한다.
8. 시스템은 선택된 입찰서의 낙찰 상태를 '낙찰'로 갱신한다.
9. 시스템은 나머지 입찰서 상태를 '미낙찰'로 일괄 갱신한다.
10. 시스템은 공고 상태를 '낙찰 완료'로 변경한다.
11. 시스템은 낙찰 공인중개사와 미낙찰 공인중개사 모두에게 결과 알림을 발송한다.
12. 시스템은 공고 상세 화면으로 복귀하여 정상 상태로 종료한다.

**Alternatives**

**A1. [3단계] 마감된 공고에 입찰서가 없는 경우**
- A1-1. 시스템은 '접수된 입찰서가 없습니다' 메시지를 표시한다.
- A1-2. 시스템은 공고 목록(3단계)으로 복귀한다.

**A2. [Cancel — 6~7단계] 매도자가 선정을 보류한 경우**
- A2-1. 시스템은 공고 상태를 '낙찰자 선정 대기'로 유지한다.
- A2-2. 시스템은 공고 목록 화면으로 복귀하며, 어떠한 상태 변경도 가하지 않는다.

**Postcondition**

- Q1. (정상 종료) 낙찰 공인중개사의 입찰서 상태가 '낙찰'이며, 공고 상태가 '낙찰 완료'인 상태이다.
- Q2. (정상 종료) 미낙찰 공인중개사의 입찰서 상태가 '미낙찰'로 갱신된 상태이다.
- Q3. (비정상 종료 — A1/A2) 경매 저장소 상태는 Precondition 시점과 동일하게 유지된다.

1. 시스템은 Validate User를 수행하여 매도자의 인증 상태를 확인한다.
2. 매도자는 경매 공고 등록을 요청한다.
3. 시스템은 매도자 소유 매물 목록을 반환한다.
4. 매도자는 공고 대상 매물을 선택한다.
5. 시스템은 선택된 매물의 기본 정보를 공고 등록 양식에 자동으로 채운다.
6. 매도자는 공고 조건(입찰 마감 기한, 요구 서비스 조건, 최소 자격 요건)을 입력한다.
7. 매도자는 낙찰 우선 기준(수수료율 가중치, 마케팅 전략 가중치)을 설정한다.
8. 시스템은 입력된 공고 조건의 유효성(마감 기한 범위, 필수 항목 충족)을 검증한다.
9. 매도자는 공고 최종 등록을 확정한다.
10. 시스템은 공고를 경매 저장소에 저장하고 공고 ID를 발급한다.
11. 시스템은 공고 상태를 '입찰 모집중'으로 설정한다.
12. 시스템은 자격 요건을 충족하는 공인중개사 목록에 신규 공고 알림을 발송한다.
13. 시스템은 매도자에게 공고 등록 완료 메시지와 공고 ID를 반환한다.
14. 시스템은 공고 목록 화면으로 복귀하여 정상 상태로 종료한다.

**Alternatives**

**A1. [4단계] 선택한 매물에 이미 진행 중인 공고가 있는 경우**
- A1-1. 시스템은 '해당 매물에 진행 중인 공고가 존재합니다' 메시지를 표시한다.
- A1-2. 시스템은 매물 선택 화면(3단계)으로 복귀한다.

**A2. [8단계] 공고 조건 유효성 검증 실패 시**
- A2-1. 시스템은 오류 필드를 강조 표시하고 수정 안내 메시지를 출력한다.
- A2-2. 매도자는 6단계로 돌아가 공고 조건을 재입력한다.

**A3. [Cancel — 6~9단계] 매도자가 공고 등록을 취소한 경우**
- A3-1. 시스템은 임시 입력 데이터를 파기한다.
- A3-2. 시스템은 공고 목록 화면으로 복귀하며, 경매 저장소에는 어떠한 변경도 가하지 않는다.

**Postcondition**

- Q1. (정상 종료) 신규 경매 공고가 경매 저장소에 저장되며, 공고 상태는 '입찰 모집중'이다.
- Q2. (정상 종료) 자격 요건을 충족하는 공인중개사에게 알림이 발송된 상태이다.
- Q3. (비정상 종료 — A1/A2/A3) 경매 저장소 상태는 Precondition 시점과 동일하게 유지된다.

### UC-02. Submit Bid (입찰 참여) — 학생 작성본

**AI 생성본 대비 변경 포인트**

| 항목 | 변경 내용 |
|---|---|
| P4 재정의 | 재제출 시 "수정 허용" 상태 조건 명시로 모호성 제거 |
| Description 5단계 | 공고의 낙찰 우선 기준 정보를 상세 화면에 포함하도록 확장 |
| Description 6단계 | 낙찰 우선 기준 참고하여 작성하도록 유도 |
| A5 신설 | 재제출(수정) 시나리오 Alternative 추가 |

| 필드 | 내용 |
|---|---|
| **Use Case Name** | Submit Bid (입찰 참여) |
| **Summary** | 공인중개사가 매도자의 역경매 공고에 대해 수수료·서비스 조건을 포함한 입찰서를 제출한다. |
| **Actor** | 공인중개사 (Agent) |
| **Dependency** | «include» Validate User |

**Precondition**

- P1. 공인중개사는 시스템에 로그인되어 인증 세션이 유효한 상태이다.
- P2. 대상 경매 공고가 '입찰 모집중' 상태이며 입찰 마감 시각 이전이다.
- P3. 공인중개사의 자격 증명(자격증 번호, 사무소 등록증)이 검증 완료된 상태이다.
- P4. 공인중개사는 해당 공고에 기제출한 입찰서가 없거나, 기제출 입찰서가 '수정 허용' 상태이다.

**Description (정상 흐름)**

1. 시스템은 Validate User를 수행하여 공인중개사의 인증 상태를 확인한다.
2. 공인중개사는 참여 가능한 경매 공고 목록 조회를 요청한다.
3. 시스템은 마감 시각이 경과하지 않은 활성 공고 목록을 반환한다.
4. 공인중개사는 입찰할 공고를 선택한다.
5. 시스템은 선택된 공고의 상세 정보(매물 정보, 매도자 요구 조건, 낙찰 우선 기준, 현재 입찰 건수)를 표시한다.
6. 공인중개사는 공고의 낙찰 우선 기준을 참고하여 입찰서(제안 수수료율, 마케팅 전략, 예상 매각 기간, 서비스 조건)를 작성한다.
7. 시스템은 입력된 입찰 정보의 유효성(필수 항목 충족, 수수료율 범위, 글자 수 제한)을 검증한다.
8. 공인중개사는 최종 입찰서 제출을 확정한다.
9. 시스템은 입찰서를 경매 저장소에 저장하고 접수 시각을 기록한다.
10. 시스템은 공인중개사에게 입찰 접수 완료 메시지와 접수번호를 반환한다.
11. 시스템은 매도자의 알림 큐에 신규 입찰 알림을 등재한다.
12. 시스템은 입찰 목록 화면으로 복귀하여 정상 상태로 종료한다.

**Alternatives**

**A1. [4단계] 선택한 공고가 이미 마감된 경우**
- A1-1. 시스템은 '공고가 마감되었습니다' 메시지를 표시한다.
- A1-2. 시스템은 공고 목록(3단계)으로 복귀한다.

**A2. [7단계] 입력 유효성 검증 실패 시**
- A2-1. 시스템은 오류 필드를 강조 표시하고 수정 메시지를 출력한다.
- A2-2. 공인중개사는 6단계로 돌아가 입찰서를 재작성한다.

**A3. [Cancel — 6~8단계] 공인중개사가 입찰 작성을 취소한 경우**
- A3-1. 시스템은 임시 입력 데이터를 파기한다.
- A3-2. 시스템은 공고 목록(3단계)으로 복귀하며, 경매 저장소에는 어떠한 변경도 가하지 않는다.

**A4. [9단계] 저장 실패 시**
- A4-1. 시스템은 처리를 취소하고 이전 상태로 복원한다.
- A4-2. 시스템은 '일시적 오류로 저장되지 않았습니다' 메시지를 반환하고 6단계로 복귀한다.
- A4-3. 시스템은 장애 로그에 실패 이벤트를 기록한다.

**A5. [4단계] 공인중개사가 기제출 입찰서를 수정 재제출하는 경우**
- A5-1. 시스템은 기존 입찰서 내용을 양식에 불러온다.
- A5-2. 공인중개사는 입찰서를 수정하고 6단계부터 정상 흐름을 이어간다.
- A5-3. 시스템은 기존 입찰서를 수정된 내용으로 갱신하고 접수번호를 유지한다.

**Postcondition**

- Q1. (정상 종료) 입찰서가 경매 저장소에 저장되며, 해당 공고(선택된 공고 기준)의 입찰 건수가 1 증가한 상태이다.
- Q2. (정상 종료) 매도자의 알림 큐에 입찰 알림 메시지가 등재된 상태이다.
- Q3. (비정상 종료 — A1/A2/A3) 경매 저장소 및 알림 큐 상태는 Precondition 시점과 동일하게 유지된다.
- Q4. (비정상 종료 — A4) 경매 저장소는 Precondition 상태로 복원되고, 장애 로그에 실패 이벤트가 1건 추가된 상태이다.

### UC-03. Select Winner (낙찰자 선정) — 학생 작성본

**AI 생성본 대비 변경 포인트**

| 항목 | 변경 내용 |
|---|---|
| Summary | 매도자 협상력 주체 강조 ("비교하고 선정") |
| Description 6단계 추가 | 낙찰 우선 기준 가중치 적용 정렬 표시 단계 추가 |
| A3 신설 | 입찰서 1건인 경우 단독 낙찰 확인 흐름 추가 |
| Postcondition Q2 | 결과 알림 발송 상태 명시 강화 |

| 필드 | 내용 |
|---|---|
| **Use Case Name** | Select Winner (낙찰자 선정) |
| **Summary** | 매도자가 마감된 역경매 공고의 입찰서를 비교하고 최종 낙찰 공인중개사를 선정한다. |
| **Actor** | 매도자 (Seller) |
| **Dependency** | «include» Validate User / «extend» Submit Bid |

**Precondition**

- P1. 매도자는 시스템에 로그인되어 인증 세션이 유효한 상태이다.
- P2. 대상 경매 공고의 입찰 마감 시각이 경과한 상태이다.
- P3. 해당 공고에 1건 이상의 입찰서가 접수된 상태이다.
- P4. 해당 공고에 대해 아직 낙찰자가 선정되지 않은 상태이다.

**Description (정상 흐름)**

1. 시스템은 Validate User를 수행하여 매도자의 인증 상태를 확인한다.
2. 매도자는 마감된 경매 공고 목록 조회를 요청한다.
3. 시스템은 입찰 마감되고 낙찰자 미선정 상태인 공고 목록을 반환한다.
4. 매도자는 낙찰자를 선정할 공고를 선택한다.
5. 시스템은 해당 공고에 접수된 입찰서 목록을 반환한다.
6. 시스템은 공고 등록 시 설정한 낙찰 우선 기준(수수료율/마케팅 전략 가중치)을 적용하여 입찰서를 정렬하여 표시한다.
7. 매도자는 입찰서를 비교 검토한다.
8. 매도자는 최종 낙찰할 공인중개사를 선택하고 낙찰 확정을 요청한다.
9. 시스템은 선택된 입찰서의 상태를 '낙찰'로 갱신한다.
10. 시스템은 나머지 입찰서 상태를 '미낙찰'로 일괄 갱신한다.
11. 시스템은 공고 상태를 '낙찰 완료'로 변경한다.
12. 시스템은 낙찰 공인중개사와 미낙찰 공인중개사 전원에게 결과 알림을 발송한다.
13. 시스템은 공고 상세 화면으로 복귀하여 정상 상태로 종료한다.

**Alternatives**

**A1. [3단계] 마감된 공고에 입찰서가 없는 경우**
- A1-1. 시스템은 '접수된 입찰서가 없습니다' 메시지를 표시한다.
- A1-2. 시스템은 공고 목록(3단계)으로 복귀한다.

**A2. [Cancel — 7~8단계] 매도자가 선정을 보류한 경우**
- A2-1. 시스템은 공고 상태를 '낙찰자 선정 대기'로 유지한다.
- A2-2. 시스템은 공고 목록 화면으로 복귀하며, 어떠한 상태 변경도 가하지 않는다.

**A3. [5단계] 접수된 입찰서가 1건인 경우**
- A3-1. 시스템은 '입찰서가 1건입니다. 단독 낙찰 처리하시겠습니까?' 확인 메시지를 표시한다.
- A3-2. 매도자가 확정하면 8단계로 이동하여 정상 흐름을 이어간다.
- A3-3. 매도자가 거부하면 A2 흐름을 수행한다.

**Postcondition**

- Q1. (정상 종료) 낙찰 공인중개사의 입찰서 상태가 '낙찰'이며, 공고 상태가 '낙찰 완료'인 상태이다.
- Q2. (정상 종료) 미낙찰 공인중개사의 입찰서 상태가 '미낙찰'로 갱신되고, 결과 알림이 참여 공인중개사 전원에게 발송된 상태이다.
- Q3. (비정상 종료 — A1/A2) 경매 저장소 상태는 Precondition 시점과 동일하게 유지된다.
